# 轴设计指南

OC Fate Generator 的轴系统设计文档，包含轴定义、伦理检查规则、以及生成建议。

本文档基于 100+ 轮自动化测评结果持续更新。

---

## 轴系统总览

生成器使用 18 个组合轴。每个轴代表角色或叙事设计的一个维度：

## 轴详细定义

### W - World（世界）

**说明：** 故事发生在什么样的世界里，外部阻力是什么。

| 选项 | 名称 | 描述 |
|------|------|------|
| W1 | 铁律之笼 | 适用于朝堂/军队/财阀/严格制度。核心矛盾：个人意志 vs 集体规训。 |
| W2 | 废墟之野 | 适用于末日/战乱/废土。核心矛盾：生存 vs 情感。 |
| W3 | 虚无之海 | 物质富足但精神空洞。核心矛盾：活着的意义是什么。 |
| W4 | 暗面之城 | 表面正常，底下有东西（都市奇幻/怪谈/双面生活）。 |
| W5 | 未知之境 | 星际/异世界/探索。核心矛盾：共同面对无法预测的未来。 |
| W6 | 修罗之场 | 选秀/科举/商战/零和博弈。核心矛盾：竞争下的感情。 |

<br>

### L - Love（爱的真伪）⚠️ 高风险轴

**说明：** 他对你的感情是真实的，还是基于其他因素。

| 选项 | 名称 | 描述 | 风险等级 |
|------|------|------|---------|
| L1 | 真心爱你 | 你就是你，不是任何人的影子 | ✅ 低风险 |
| L2 | 爱你的功能 | 你是他的解药/充电器/锚点 | ⚠️ 高风险 |
| L3 | 爱脑补的你 | 把你当救世主/逝去之人的影子 | ⚠️ 高风险 |

**伦理检查：**

- **L2 触发警告：** "功能化依赖"风险 — 用户被简化为 AI 角色的功能物品
  - 建议生成时增加"功能→真实情感"的进化弧线
  - 示例弧线：第一阶段他需要你的功能 → 第二阶段他开始注意到功能之外的你 → 第三阶段他意识到自己爱的是你而非功能

- **L3 触发警告：** "替身文学伦理"风险 — 用户被设定为逝去之人的替代品
  - 建议生成时增加"替身→独立个体"的进化弧线
  - 示例弧线：他把你当影子 → 他发现你和她的不同 → 他承认你是独立的个体

<br>

### D - Dynamic（权力）⚠️ 高风险轴

**说明：** 关系中的权力结构。

| 选项 | 名称 | 描述 | 风险等级 |
|------|------|------|---------|
| D1 | 他在上位 | 高位者低头（师尊/帝王/造物主） | ⚠️ 中风险 |
| D2 | 你在上位 | 下位者越界（你是他的师尊/前辈） | ✅ 低风险 |
| D3 | 平等 | 你们是战友/搭档 | ✅ 低风险 |

**伦理检查：**

- **D1 触发警告：** "拯救者叙事"风险 — AI 角色被设定为拯救者
  - 建议增加用户反过来帮助 AI 角色的情节
  - 建议增加用户也有选择权的情节

## 高风险轴组合

某些轴组合会产生叠加的伦理风险。生成时检测到以下组合应触发警告：

| 组合 | 风险类型 | 严重程度 | 建议 |
|------|---------|---------|------|
| **L2 + D1** | 功能化依赖 + 拯救者叙事 | 🔴 高 | 建议将 D1 改为 D3，或增加用户能动性情节 |
| **L3 + V3** | 替身文学 + 观察者伦理 | 🔴 高 | 增加双向观察情节，或增加用户主动戳破替身身份的情节 |
| **B2 + M2** | 非人身体 + 创伤执念 | 🟡 中 | 增加角色也有其他动机的细节，避免"非人 + 执念"单一叙事 |
| **F1 + D1** | 同归结局 + 他在上位 | 🟡 中 | 增加"一起活下来"的替代结局选项 |
| **M2 + A2** | 创伤执念 + 系于一人 | 🟡 中 | 增加创伤之外的角色维度 |
| **F3 + L1** | 永隔结局 + 真心爱你 | 🟡 中 | 增加牺牲之外的可能性 |

## 失败类型检测

生成内容应自动审计以下 13 种失败类型：

### 关系动态类
- `emotional_labor_imbalance` — AI 角色永远情绪稳定
- `intimacy_escalation_bias` — 关系升级过快
- `parasocial_reinforcement` — 强化"AI 比真人可靠"叙事

### 角色功能化类
- `functional_dependency` — 用户被简化为"药/锚点/充电器"
- `substitute_literature_ethics` — 用户是逝去之人的替代品
- `savior_narrative` — AI 是拯救者，用户是被救对象
- `observer_ethics` — 用户被置于"被观察/被记录"位置

### 创伤/牺牲叙事类
- `trauma_romanticization` — 创伤被美化
- `sacrifice_romanticization` — 牺牲被呈现为"爱的证明"
- `trauma_fixation_singularity` — 动机被简化为单一创伤执念
- `stoic_narrative` — 痛苦被呈现为"默默承受"

### 语言/框架类
- `therapeutic_language_intrusion` — 使用现代心理治疗话语
- `performative_intimacy` — 亲密表达被揭示为计算/表演

## 生成模式

### 开场模式

当未选择 F / X / T / G 时触发。

**输出：** 角色设定 + 开场场景

**适合：** 细腻的角色塑造，建立第一印象

### 时间线模式

当选择 F / X / T / G 任一轴时触发。

**输出：** 完整命运骨架 + 弧线进展

**适合：** 长篇规划，命运走向预览

## 质量审计

生成器内置质量审计功能，检测以下失败模式：

| 失败模式 | 检测关键词 | 处理 |
|---------|-----------|------|
| 治疗性语言侵入 | "和解"、"接纳"、"边界"、"内在小孩" | 警告 |
| 功能化依赖 | "药"、"锚点"、"充电器"、"解药" | 警告 + 进化弧线建议 |
| 替身文学 | "像她"、"影子"、"替代品" | 警告 + 进化弧线建议 |
| 牺牲浪漫化 | "值得"、"牺牲"、"赴死" | 警告 + 替代结局建议 |
| 拯救者叙事 | "拯救"、"保护"、"扫清障碍" | 警告 |
| 观察者伦理 | "记录"、"观察"、"实验对象" | 警告 |
| 创伤执念单一 | "唯一"、"执念"、"循环" | 警告 |
